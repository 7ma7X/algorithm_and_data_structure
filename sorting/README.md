## バブルソート

O(n^2)

## クイックソート

O(n logn)  
理想的な場合には、QuickSortmが呼ばれるたびに問題が半分の大きさになるので、全体ではO(n logn)  
最悪の場合には、分割のたびに1:n-1で分割されるので、全体ではO(n^2)  
平均するとO(n logn)

## クイックセレクト

クイックソートの考え方を用いて小さいものからn番目の要素を探す

## マージソート

クイックソートと同様分割統治を利用する。  
機械的に列を2つに分解するので、平均でも最悪でも O(n logn)  
メモリ上に収まりきらない巨大なファイル上のデータをソートする**外部ソート**で有用
